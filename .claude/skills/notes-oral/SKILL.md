# Skill : Ajouter une note pour l'oral

## Objectif
Ajouter dans `docs/NOTES-ORAL.md` un point que Tony doit maîtriser pour l'oral, identifié pendant la session de rédaction.

## Quand utiliser ce skill
- Tony pose une question qui montre qu'il ne maîtrise pas un concept (ex : "c'est quoi l'architecture 3 tiers ?")
- Tony s'interroge sur un choix technique (ex : "pourquoi pas de Repository pattern ?")
- Un point est abordé que le jury pourrait demander d'expliquer
- Une distinction importante est clarifiée (ex : enum vs state machine, HttpOnly vs blacklist)

## Comment ajouter la note
1. Lire `docs/NOTES-ORAL.md` pour identifier la bonne section
2. Ajouter la note dans la section thématique existante, ou créer une nouvelle section si nécessaire
3. Format de chaque note :
   - **Titre du concept** en gras
   - Explication concise (2-5 lignes) en langage simple
   - Si pertinent : ce que Tony doit répondre au jury

## Règles
- **Ne pas dupliquer** — vérifier que la note n'existe pas déjà
- **Rester concis** — c'est un aide-mémoire, pas un cours
- **Langage simple** — Tony doit pouvoir relire et comprendre rapidement
- **Ancrer dans GreenRoots** — toujours relier au projet concret
