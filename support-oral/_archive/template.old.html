Architecture

Honnêteté sur les choix d'architecture

Choix assumé Architecture modulaire en couches (Controller → Service → Prisma), pensée pour la testabilité via l'injection de dépendances.
Nuance Ce n'est pas une architecture hexagonale au sens strict : le service dépend directement de Prisma - choix pragmatique pour 4 sprints.
Risque neutralisé Broken Access Control (OWASP A01) : double contrôle rôle (guard) + propriétaire (order.userId === userId).
Assumer les choix, montrer qu'on connaît les alternatives = posture d'ingénieur mature.
